## 1. Summary

A single-PDA vault custodying one SPL/Token-2022 mint. Users `deposit` tokens (credited to a
running total) and `withdraw` them. The vault PDA signs token transfers out. Core trust
assumption (target state): only the recorded `authority` can withdraw, and `total_deposited`
never exceeds the vault's real token balance.

## 5. Token Model

- Token program(s): **SPL Token + Token-2022** via `Interface<TokenInterface>` + `InterfaceAccount`.
- Vault does not issue a token (no mint/freeze authority to manage here).
- Token-2022 extension policy: credit **actual balance delta** on deposit (transfer-fee safe);
  for value-critical correctness, allow-list mints or reject transfer-hook mints (branch 8.4).
- Transfers use `transfer_checked` (decimals verified).

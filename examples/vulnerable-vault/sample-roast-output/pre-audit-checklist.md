## Accounts & PDAs
- [x] Data accounts read via typed `Account<T>` (owner + discriminator checked) — `lib.rs` (Vault is typed)
- [ ] PDAs use canonical bump, no user-supplied bump — **F-05**: `initialize` takes `vault_bump`
- [-] PDA cross-domain sharing — single PDA, single role
- [ ] Account relationships enforced (`has_one`) — **F-01**: `withdraw` has no `has_one = authority`
- [-] Duplicate-mutable-account cases — no same-type writable pairs
- [x] `init` used (not `init_if_needed`) — `Initialize` uses `init`

## Authority & Signers
- [ ] Every privileged account is a `Signer` — **F-01**: `withdraw.authority` is `UncheckedAccount`
- [ ] Each signer bound to its resource — **F-01**: not bound to `vault.authority`
- [x] Admin vs. user privilege separated — only `initialize` is admin
- [-] Two-step authority transfer — no authority-transfer instruction
- [x] Access matrix exists — design-spec §4
